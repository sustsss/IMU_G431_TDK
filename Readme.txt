24/6/24
1.此版本改进了落地后可能因为撞击再次触发传输数据的问题
    方案是增加一个计时器，一次触发后要等6s才能进行下一次触发，就能避免上述问题
2.此版本解决了yaw值漂移的问题。当经过较长时间后yaw值相对于初始值会漂移。
    方案：将要开始传递时yaw的值设为参考值，然后用yaw值减去参考值便是存入sd卡的值。此办法能够解决yaw漂移的问题，但是也存在隐患。
    例如：当参考值为-178时，那假如yaw值为180,则存入sd的值为358,显然是不合理的。但是考虑到yaw的变化应该是缓慢的，所以不会出现前面的情况。因此在测试的情境下此办法是可行的。
3.此版本解决了当roll值偏转超过90时传输数据会出错的情况。
    判断方法是 abs（roll）>70时，即认为后面的数据便直接不能用了，直接舍弃。就能避免上述的问题。

24/6/25
1.此版本解决了yaw值会有突变的情况
   上版本是存在问题的，因为陀螺仪yaw值的范围为（-180,180），当穿越180或者-180时，值由一个极端变到另一个极端。假如y_Init的值在极端值附近时，存入sd卡的值便会发生突变。
此版本加了一些判断，只要记录数据期间陀螺仪yaw的变化值不超过180度就不会存在问题。
